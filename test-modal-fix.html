Modal Click Handler Fix - Before & After Comparison

Demonstrating the issue and the solution for non-responsive modal close buttons

‚ùå BEFORE (Broken)

ISSUE: Clicks don't close modal

Problem: Backdrop handler calls onClose directly, but buttons also trigger backdrop click through event bubbling.

<div onClick={onClose}> <button onClick={onClose}> Close </button> </div>

Why it fails: When button is clicked, event bubbles to backdrop, causing onClose to be called twice, which might cancel each other out.

‚úÖ AFTER (Fixed)

FIXED: All clicks work correctly

Solution: Check e.target === e.currentTarget in backdrop handler and use e.stopPropagation() in buttons.

const handleBackdropClick = (e) => { if (e.target === e.currentTarget) { onClose(); } }; const handleCloseClick = (e) => { e.stopPropagation(); onClose(); }; <div onClick={handleBackdropClick}> <button onClick={handleCloseClick}> Close </button> </div>

Why it works: Backdrop only closes when clicking the backdrop itself, not when clicks bubble up from child elements.
